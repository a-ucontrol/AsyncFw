Detailed Description

Asynchronous mDNS/DNS-SD service responder and discovery browser.

Multiple instances can run independently with different service types.

◆ QuerierMode

enum AsyncFw::MulticastDns::QuerierMode : uint8_t

Operation modes for the network query browser.

Defines how the discovery queries are broadcasted, which local ports are utilized, and how remote nodes are expected to respond.

Enumerator
Multicast 

Standard mDNS multicast mode.

Binds to the standard mDNS port (5353). Queries are sent via multicast, and responses from remote nodes are also broadcasted via multicast to the entire network. Allows full caching by all peers but increases network traffic.

Unicast5353 

Hybrid unicast mode via the standard port.

Binds to the standard mDNS port (5353). Queries are sent via multicast, but remote nodes are requested to send their responses via Unicast directly to this node. This minimizes network noise while still allowing the socket to passively capture unsolicited background Announce and Goodbye events from peers.

Unicast 

Legacy Unicast mode via an ephemeral port.

Binds to a random, OS-assigned ephemeral port (0). Queries are sent via multicast, and responses are returned directly via Unicast. Useful as a failover when port 5353 is locked by another system daemon, but cannot capture unsolicited background multicast announcements or goodbyes.

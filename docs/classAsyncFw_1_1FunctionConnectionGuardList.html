Detailed Description

A helper container designed to manage multiple FunctionConnectionGuard lifecycles at once.

Inherits from std::vector<FunctionConnectionGuard>. It provides a convenient way to aggregate multiple connection guards within a single scope (e.g., inside a controller or view class). When the list goes out of scope, all registered connections are safely and automatically disconnected.

Example:

class Controller {
AsyncFw::FunctionConnectionGuardList connections;
public:
void init(Sensor &sensor) {
// Conveniently append connection guards using the += operator
connections += sensor.valueChanged.connect(this, &Controller::onValueChanged);
connections += sensor.statusChanged.connect(this, &Controller::onStatusChanged);
}
// All connections are severed automatically when Controller is destroyed
};

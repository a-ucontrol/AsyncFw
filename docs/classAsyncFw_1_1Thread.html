◆ destroying

FunctionConnector ::Policy<AbstractFunctionConnector::Direct>::Protected<Thread> AsyncFw::Thread::destroying

Signal emitted inside the destructor framework immediately before the thread object resources are wiped.

Warning
Defaults to a Direct connection policy. If explicitly overridden to an asynchronous or Queued connection, ensure the subscriber lambda does NOT capture or reference the this pointer of the dying Thread instance to avoid use-after-free conditions.

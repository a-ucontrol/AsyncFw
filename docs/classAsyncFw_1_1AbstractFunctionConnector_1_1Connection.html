Detailed Description

Represents an active linkage between a sender's signal/connector and a specific receiver slot.

The Connection class representing a single active subscription.
It stores crucial metadata required for dispatching events, including the target execution context AsyncFw::AbstractThread and the requested synchronization strategy Type.
Instances of this class are typically allocated on the heap when a subscriber calls connect. Lifetime and memory cleanup are managed automatically by the framework, or can be tied to scopes via FunctionConnectionGuard.

enum AsyncFw::AbstractFunctionConnector::Connection::Type : uint8_t

Specifies how a particular receiver slot should be invoked relative to the sender's thread.

Defines the precise execution context and thread synchronization semantics for a specific connection instance during event dispatching.

Enumerator
Auto 

Evaluates the context at runtime. Uses Direct if the sender and receiver share the same thread ID, otherwise uses Queued.

Direct 

Invokes the receiver slot immediately inside the sender's thread. No context switching occurs.

Queued 

Posts a task containing a copy of the arguments into the receiver thread's event loop. Execution is asynchronous.

Sync 

Dispatches the invocation to the receiver's thread loop, but blocks the sender's thread until execution completes.

Default 

Falls back to the default ConnectionPolicy configuration defined by the parent connector instance.
